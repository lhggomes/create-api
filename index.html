2 Entendo um pouco da Estrutura REST

Uma API REST, constitui basicamente da implementao da estrutura REST j disponvel no protocolo HTTP, tal como:

         GET: Utilizado para capturar dados da nossa API.

         POST: Utilizado para enviar informaes para a nossa API.

         PUT: Utilizado para enviar atualizaes de dados em nossa API.

         DELETE: Utilizado para deletar informaes na nossa API.

 

Atravs dos mtodos que so disponibilizados, possvel criar uma interao com diversos sistemas.

3 Criao Modelos

Como vamos utilizar a estrutura da Java Persistence API, vamos criar modelos (exemplificao) das tabelas do nosso banco de dados na linguagem Java. Para isso, vamos utilizar o pacote Model da nossa estrutura.

Para cada modelo, iremos criar uma nova classe, vamos dar uma olhadinha como ficar:

Alguns pontos importantes, que podemos verificar na construo do cdigo, alguns pontos importantes, so eles:

1.        @Entity: Precisamos desta anotao, que provm da JPA, para informar que essa classe Java, tambm ir ser uma representao de um banco de dados.

2.       @Id: Como toda representao em um banco de dados, devemos informar qual campo ser a chave primria. Essa anotao, ser utilizada tambm junto com a anotao @GeneratedValue, que ir informar para o JPA qual ser a estratgia utilizada para criao da chave primria.

3.       @Column: Essa anotao, fornecida, pelo JPA, para manuteno de campos, que como esto em nossa entidade, sero colunas no nosso banco de dados. Podemos notar que a propriedade unique est como True, isso informa para o JPA, que a coluna no poder receber valores repetidos.

 

Antes de verificarmos a ltima anotao disponvel no nosso cdigo, importante ressaltar os tipos de relacionamento que podem existir entre Classes (Entidades).

         @ManyToMany;

         @ManyToOne;

         @OneToOne;

 

Elas so responsveis por implementar a estrutura do modelo de entidade e relacionamento do nosso projeto.

Em nosso projeto, estamos assumindo que um usurio pode ter diversos endereos, logo, utilizaremos a anotao @ManyToMany.

 

4.      @ManyToMany: Essa anotao utilizada para referenciar uma outra entidade no banco de dados, que possua vez, ser feito o relacionamento. Podemos notar tambm, que ele receber o parmetro mappedBy, esse parmetro responsvel por informar para JPA, que o controle dos endereos, ser mapeado pela classe de Usurio.

 

Acima, podemos verificar a implementao da classe de Endereo, podemos notar que os termos utilizados na sua implementao so semelhantes nossa classe de usurios. O que muito bacana, o JPA nos permite manter uma estrutura semelhantes para nossas classes.
